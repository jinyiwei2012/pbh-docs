# 任务反吸血

任务反吸血模块为用户提供常见的不良客户端屏蔽功能。

## 触发条件

- **管道**：种子处于[活跃传输状态](../faq.md#为什么配置了封禁规则但有些-peer-没有被封禁)
- **前置**：预设已启用（`presets.xunlei.enabled = true`）
- **缓存**：无缓存
- **匹配**：PeerID/ClientName 匹配迅雷特征

| 条件 | 动作 |
|---|---|
| 迅雷非 0019 版 | **BAN**（任何任务） |
| 迅雷 0019 版 + 做种中 | **BAN** |
| 迅雷 0019 版 + 下载中 | 放行 |

### 迅雷

考虑到有大量的迅雷用户存在，本模块旨在保持 BitTorrent 的以牙还牙精神的基础上，尽可能沟通迅雷和下载器双方用户进行有效的用户传输。具体规则策略如下：

1. 如果迅雷是 0019 版本：允许迅雷客户端连接到用户用户客户端上正在下载的任务上，但禁止连接到用户客户端上的正在做种的任务。这样做的原因是迅雷在下载完成后不会在 BT 网络上做种，但在下载过程中会和 BT 网络用户分享上传资源
2. 如果是其它版本迅雷（如 0012、0018）：禁止迅雷客户端连接到用户客户端上的任何任务，因为这些客户端不能正确汇报任务进度，且大部分构建都拒绝上传分享资源。这与 BitTorrent 精神相违背，且对网络没有任何正向作用。

本模块所述描述并非一成不变，在未来如果迅雷对上传做出了调整，本策略也会随之更新。
